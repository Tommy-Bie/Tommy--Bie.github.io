Git Notes

在搭建博客以及使用github时都接触到了Git这个强大的分布式管理工具，所以学习了廖雪峰老师的教程并做了笔记，以便自己以后随时查看

创建Git仓库

git的命令操作主要是在 git bash 终端进行的

使用 git init 命令将当前目录变为Git可以管理的仓库

比如我们将learngit目录变为Git仓库：

使用 ls -ah命令可以看到当前面目录多了一个 .git 目录，用于Git跟踪管理版本库

假如我们在 learngit 中创建一个 readme.txt 并写了两行内容

Git is a distributed version control system.
Git is free software.

将修改提交到Git版本库

之后我们使用两步将该txt文件加入到Git仓库

  1. git add <file>
  1. git commit <file>

git commit -m “…” # 其中…使我们自己写的说明

Git 告诉我们有一个文件改动了（即readme.txt），且插入了两行内容

注：Git 使用两步 add 和 commit 的方法来添加文件是因为 commit 可以一次提交很多文件，所以我们可以add多个文件，并只需要一次 commit 就能全部上传到 Git 仓库

现在继续修改 readme.txt ：再第一行中加入 distributed

Git is a distributed version control system.
Git is free software.

使用 git status 命令查看当前状态

使用 git diff 命令可以看到具体修改的内容

然后现在使用 add 命令

再使用 commit 正式提交之前 使用 git status 看看 add 之后的情况

使用 commit 命令提交到 Git 版本库

此时用 git status 查看状态可以发现当前已经没有需要提交的修改了 且work tree clean

工作区(Working Directory)、暂存区(Stage)和分支(Branch)

我们之前所使用的的 learngit 文件夹就是一个工作区

Git版本库就是工作区中的一个隐藏文件 .git ，里面存了很多东西，主要是暂存区(Stage或index)和Git为我们自动创建的第一个分支(branch)，该分支名叫master ，指向 master 的指针叫HEAD

而我们之前所使用的 git add 就是将文件放入了暂存区，而 git commit 就是把所有存放在暂存区的文件送入当前分支

我们创建Git版本库时，Git为我们自动创建了第一个分支master， 所以我们 git commit 就是上传到master分支中

修改管理

需要注意的是，Git管理的是修改而不是文件，加入你第一次修改文件并add之后，再进行了第二次修改，那么此时commit只会把第一次修改的内容上传到master分支中，因为此时暂存区中只有第一次修改的内容，而第二次修改的内容还未存入暂存区。

并且可以通过git diff HEAD -- <file> 命令来比较工作区和版本库里面最新版本的区别

当你对文件进行了一次修改但又想改回来的时候（还没add），可以在使用 git checkout -- <file>来丢弃工作区的修改

或者使用git restore <file>来丢弃修改

若是已经add了（即修改已经在暂存区了），可以使用git reset HEAD <file>将暂存区的修改撤销掉，重新放回工作区

此时暂存区已经干净了，刚刚的修改位于工作区（此时再使用 checkout 或 restore就可以清除掉工作区的修改）

注：git reset既可以用于版本回退，又可以将暂存区的修改回退到工作区

文件删除

假设你的版本库中有某个文件（即该文件被add且commit给版本库了），你现在想要删除这个文件：当你手动将文件删除或者使用rm <file>删除文件，git status都能告诉你哪些文件被删除了，接下来有两种情况：

1）你想彻底将该文件删除，即版本库中也不留下，那么先git rm <file>再git commit -m"..."（这和先add再commit思路差不多）：

2）你的文件是被你手动误删了，那么由于该文件还在版本库中，可以使用git checkout -- <file>将工作区恢复到版本库中的版本
